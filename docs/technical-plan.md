# Technical Plan

## Стек (пропозиція)
- `HTML + CSS + JavaScript` без фреймворків на старті.
- Причина: швидкий запуск, мінімальна складність, легко кастомізувати під мобільні жести.

## Layout
- `body, html`: `height: 100%`, `overflow: hidden`.
- Кореневий контейнер: `height: 100dvh` (fallback: `100vh`).
- Сітка:
  - `header`: фіксована висота;
  - `main`: займає весь залишок.

## Стан застосунку
- Єдиний об'єкт стану:
  - `mode: "classic" | "beads"`
  - `round: number`
  - `countInRound: number` (0..107)
  - `totalCount: number` (опціонально, можна обчислювати)

## Логіка інкремент/декремент
- Винести у чисті функції `increment(state)` / `decrement(state)`.
- Окремо тестувати граничні випадки:
  - `107 -> + -> round+1, count=0`
  - `round>1 && count=0 -> - -> round-1, count=107`
  - `round=1 && count=0 -> - -> без змін`

## Режим Classic
- Кнопка `+` викликає `increment`.
- Кнопка `-` викликає `decrement`.
- Дисплей показує `countInRound + 1` або `countInRound` залежно від фінального UX-рішення.

## Режим Beads
- Вертикальна колонка декоративних бусин (CSS/DOM).
- Обробка жестів через `Pointer Events`:
  - `pointerdown`: запам'ятати стартову координату `Y`.
  - `pointerup`: порівняти з поточною `Y`.
  - якщо `deltaY > threshold` (тягнули вниз) => `increment`.
  - якщо `deltaY < -threshold` (тягнули вгору) => `decrement`.
- `threshold` (наприклад 24-36px), щоб відсіяти випадкові рухи.
- Для запобігання скролу: `touch-action: none` на інтерактивній зоні.

## Адаптивність
- Mobile first.
- Breakpoints:
  - `>= 768px`: більша типографіка, центрування контейнера.
  - `>= 1024px`: обмеження максимальної ширини, щоб не розтягувати UI.

## Етапи реалізації
1. Каркас сторінки та блокування скролу.
2. Базова логіка лічильника і раундів.
3. Режим `Classic`.
4. Режим `Beads` + жести.
5. Полірування UX, анімації, persist (за потреби).
